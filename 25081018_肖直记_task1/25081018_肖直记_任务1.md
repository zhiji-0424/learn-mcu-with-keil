# 基础知识

**流水灯**：几个灯依次点亮，有固定数量的灯保持亮起。

**跑马灯**：只有一个灯点亮并实现“移动”。

**IO口操作**：IO口指定输出0（低）电平时可等效为该口接地（负极），输出1（高）电平可等效为该口接入一定电压的正极，切换方式我暂时不了解。

**软件延时**：执行一条指令需要时间，执行一定次数的空循环可实现延时，此时使用的是软件逻辑方式实现。时钟控制单片机的运行，根据时钟频率可推算出需要的循环次数（同一时刻只运行一个程序的情况下）。暂时不了解时钟的具体运行原理。

**按键抖动**：金属弹片按键在按下时因为弹性会弹动，类似小球掉落的上下运动。消抖方法：初次检测到按下时，等弹跳运动结束再此检测按键状态。

**呼吸灯**：在总时间一定的情况下，调整灯亮和灭的持续时间。

# 实现思路

按键各个模块的切换。（本次任务没有对按键消抖）

# 实现逻辑

基础部分全部完成，扩展部分完成2、3。

要实现的模块：

1. 流水灯
2. 跑马灯
3. 线性呼吸灯
4. 二进制显示

按键作用分配：

1. 控制时间间隔/二进制值增加，根据当前展示的模块决定按键作用。
2. 切换展示的模块。
3. 控制流水灯个数（按键计数增加，范围1~4），长按切换呼吸灯线性或非线性。
4. 在流水灯或跑马灯过程以及呼吸灯过程中实现暂停。

# 问题

1. 语法错误（函数名称、预处理指示拼写错误）有警告，但仍然可能生成程序文件。
2. 有的按键会发生抖动，有的按键抖动概率低。
3. 流水灯和跑马灯任务中，若直接使用延时，则随着延时变长，可能无法及时检测按键的状态，导致按键无反应。一种解决方法：使用“软件计时器”，1ms（可接受的延迟）计数加一，16位无符号数字（0~65535）可表示到65秒。在这里，int占内存16位：2字节。
