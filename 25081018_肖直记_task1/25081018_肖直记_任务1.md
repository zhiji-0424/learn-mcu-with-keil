# 基础知识

**流水灯**：几个灯依次点亮，有固定数量的灯保持亮起。

**跑马灯**：只有一个灯点亮并实现“移动”。

**IO口操作**：IO口指定输出0（低）电平时可等效为该口接地（负极），输出1（高）电平可等效为该口接入一定电压的正极，切换方式我暂时不了解。

**软件延时**：执行一条指令需要时间，执行一定次数的空循环可实现延时，此时使用的是软件逻辑方式实现。时钟控制单片机的运行，根据时钟频率可推算出需要的循环次数（同一时刻只运行一个程序的情况下）。暂时不了解时钟的具体运行原理。

**按键抖动**：金属弹片按键在按下时因为弹性会弹动，类似小球掉落的上下运动。消抖方法：初次检测到按下时，等弹跳运动结束再此检测按键状态。

**呼吸灯**：在总时间一定的情况下，调整灯亮和灭的持续时间。

# 问题

1. 语法错误（函数名称、预处理指示拼写错误）有警告，但仍然可能生成程序文件。
2. 有的按键会发生抖动，有的按键抖动概率低。
3. 流水灯和跑马灯任务中，若直接使用延时，则随着延时变长，可能无法及时检测按键的状态，导致按键无反应。一种解决方法：使用“软件计时器”，1ms（可接受的延迟）计数加一，16位无符号数字（0~65535）可表示到65秒，每次切换模块展示会重置计数器已延长使用时间。在这里，int占内存16位：2字节。使用软件计时器，实际时间受运行其他指令的影响，如控制语句的条件判断、跳转等，实测2.5秒
4. 有符号类型的二进制位左右移动，若是负数，右移时符号位会自动保留。如：1000 -> 1100。无符号时，则为：1000 -> 0100。AI：C 标准仅固定无符号数右移为逻辑右移，有符号数右移是 “实现定义”—— 具体是算术还是逻辑右移，取决于所用编译器，需参考编译器手册确认。“<<”总是逻辑左移（低位补0），对无符号数：只要移位位数有效，就安全，对有符号数：左移容易触发未定义行为。移位位数必须严格小于类型位宽且非负。
5. 在非线性流水灯中，若使用正弦函数，则会出现计算耗时太长（可能？），导致LED亮度变化不够平滑的问题。
